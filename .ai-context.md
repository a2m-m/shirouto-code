## 5. Known pitfalls

- `package-lock.json` の diff が大きく diff size warning が出るが、初回 npm install のため想定内
- `terminal-sidecar-requirements.md` がリポジトリに存在しない（Issue #2 の参考文書）。後続 Issue 実装時に内容を確認すること
- `onDidWriteTerminalData` は proposed API。本番動作には VS Code を `--enable-proposed-api a2m-m.shirouto-code` で起動する必要がある。Issue #4 実装時に注意
- Issue #4（PTY）は `ExtensionTerminalOptions` を使う。Issue #3 の `managedTerminal` 追跡ロジックを流用できる ✅ 実装済み
- `managedTerminal` は Terminal Profile（Issue #3）と PTY（Issue #4）で共有。PTY 起動時に既存セッションがあれば再利用する設計（後続 Issue でセッション種別の識別が必要になる可能性あり）
- `node-pty` は C++ ネイティブモジュール。VS Code の Electron ABI に合わせて `npm run rebuild` が必要。CI でも `electron-rebuild` ステップを追加すること（Issue #47）
- `ZshHookInjector` の `preexec` で取得するコマンド文字列はシェルが展開する前の文字列。エイリアスは展開されない
- PTY セッションのマーカー検知は `activePty !== undefined` のときのみ有効（Shell Integration が有効な環境との二重発火を防ぐ）
