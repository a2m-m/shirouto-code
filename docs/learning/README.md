# Learning by Commit Guide (LCG) の学習資産

このディレクトリは、日々のコミット差分（Diff）から得られた学びを蓄積し、チーム全体の知見へ変換するための資産保管場所です。

## 目的
* **Guardrail teaches humans:** 仕組みのガードが学習を生むという OS の原則に基づき、CI/Guardrail の指摘や深いコードレビューの議論を教材化する。
* **小さな資産化:** 同じミスや設計の落とし穴をチームで共有し、継続的にチームの筋肉（開発力・レビュー力）を増やす。

## 運用方針
1. 学習のトリガー: 日々のコミットや PR の更新、または CI ワークフローでの LCG トリガー（`lcg.yml` 等）によって、有意義な差分（設計的気付きやアンチパターンの回避につながる過去のコード変更）を抽出します。
2. 出力フォーマット: 各資産ファイル (`<commit-hash>.md`) は以下の構成を標準とします。
   - **Summary**: 学びの要点（短く）
   - **Deep dive**: 設計や実装に関する深い考察（最大 N 項目）
   - **Quiz**: 理解を確認するためのクイズ（最大 3 問）
   - **Explore next**: 次に見るべき観点や関連資料
3. 保存場所: 抽出された学びは資産として `docs/learning/commits/<commit-hash>.md` 等の形で保管されます。
4. 活用: 今後の Pull Request や新機能開発時、類似の課題に直面した場合にエージェントや開発者が素早く参照できる知識ベースとして機能します。
